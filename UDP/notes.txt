Go-Back-N (GBN)
- the sender is allowed to transmit pultiple packets without waiting for an acknowledgement, but is constrained to have 
no more thatn some allowable number N unacknowledged packets in the pipeline
- base > sequence number of the oldest unacknowledged packet
- nexseqsum to be the smallest unused sequencenumber (next to be used)
- four intevals in the range of sequence numbers can be identifiet
    - interval [0,base-1] packets that have been transmitted and acknowledged
    - [base, nextseqsum-1] packets that have been sent, but not acknowledged
    - [nextseqsum, base+N-1] used for packets that can be sent immideately, should data arrive from the upper layer
    - seq >= base + N cannot be used until an unacknowledged packet currentylu in the pipeline has been acknowledged

- N > window size
- why? flow control, tcp congestion control

- packets seq num is carried in packet header. if k is th enumber of bits in the seq number field, the range of 
sequence numbers is thus [0, 2k-1]

sender
- invocation from above. check if the window is full. if not > packet is sent. if full > back to the upper layer / buffered
- reciept of an ack. cumulative acknowledgement, indicating that all packets with sequence number up to n have been correctly received
- timeout event. lost or delayed packet? if timeout > sender resends ALL PACKETS! that have been sent but not acknowledged


receiver
- if packet with seq n is received correctly and in order > send ack for packet n + deliver data
> all other cases: discard packets. resend ack to the most recent received packet. 
- recall all data has to be in order to hte upper layer!
- only information is the seq num of the next in-order packet expectedseqnum
